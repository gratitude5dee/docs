Zap lets you define reusable generative video recipes in a single `Zap.md` file, then run them with zero spend in plan mode or against real AI providers with a single flag. The `@wzrdtech/zap` CLI scaffolds projects, validates recipes, and orchestrates multi-step pipelines across GMI Cloud, fal, Prodia, and Runware — all enforced by per-recipe budget caps.

## How It Works

<Steps>
  <Step title="Scaffold a project">
    Run `npx @wzrdtech/zap@0.2.0 init my-app` to create a Zap project with the correct directory structure, package scripts, and a sample recipe.
  </Step>
  <Step title="Author a recipe">
    Use `zap new <slug>` to scaffold `agent/skills/zap-<slug>/Zap.md` with YAML frontmatter defining inputs, pipeline steps, budget cap, and output format.
  </Step>
  <Step title="Validate and lint">
    Run `zap validate` to check frontmatter correctness and variable references. Run `zap lint` to catch policy warnings like live provider defaults.
  </Step>
  <Step title="Plan or run live">
    Execute `zap run <Zap.md>` for a zero-cost plan, or add `--live` to submit real provider jobs — budget is enforced automatically before any spend.
  </Step>
</Steps>

## Key Features

<CardGroup cols={3}>
  <Card title="Recipe Files" icon="scroll">
    `Zap.md` combines YAML pipeline metadata with human-readable context in a single Markdown file that agents and creators both understand.
  </Card>
  <Card title="Plan by Default" icon="flask">
    Every CLI run defaults to plan mode — schema validation and priced quotes with zero provider spend. Add `--live` only when you're ready to dispatch.
  </Card>
  <Card title="Budget Enforcement" icon="shield-check">
    Per-recipe `cap_usd` is enforced at the CLI and web runtime before any provider job is submitted.
  </Card>
  <Card title="Multi-Provider BYOK" icon="shuffle">
    Route individual steps to GMI Cloud, fal, Prodia, or Runware — all bring-your-own-key, with retry and fallback per step.
  </Card>
  <Card title="Eve Agent Runtime" icon="robot">
    Skill directories are portable agent capabilities. Coding agents can read, edit, validate, and run recipes through the same CLI.
  </Card>
  <Card title="Web Studio" icon="display">
    The creator studio at `/studio` gives a chat-style interface to run and iterate on Zaps without touching the command line.
  </Card>
</CardGroup>
